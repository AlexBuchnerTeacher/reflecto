# Security Policy

## Supported Versions

Aktiv gepflegt: aktueller `main`-Branch und die letzte Release‑Version (`v*.*.*`).

## Melden von Schwachstellen

- Erstelle ein privates Security‑Advisory in GitHub (Security → Advisories) oder kontaktiere den Maintainer über Issues mit dem Label `type:bug` und dem Präfix "[SECURITY]".
- Bitte keine Exploits öffentlich posten, bevor ein Fix bereitsteht.

## Offenlegung

- Wir bestätigen den Eingang, prüfen die Meldung und planen einen Fix.
- Nach dem Release werden Details transparent dokumentiert.
